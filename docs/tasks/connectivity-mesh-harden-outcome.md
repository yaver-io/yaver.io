### TestFlight — BLOCKED on cold-start tooling

`./scripts/deploy-testflight.sh` bumped `CFBundleVersion` 447 → 448,
started archiving, and failed with `'Yaver.xcworkspace' does not exist`.
This worktree has no `mobile/ios/Pods` or `Yaver.xcworkspace` — those are
generated by `expo prebuild` + `pod install` and are gitignored. A
first-time cold-start rebuild is ~28 min for pods alone plus ~15-20 min
to archive — safely outside a single autorun iteration's wall clock.

`expo prebuild --platform ios --no-install` immediately errors on
`ios/Yaver/Supporting/Expo.plist` missing (needs `--clean`, which then
wipes the force-tracked overlays and requires `git checkout -- mobile/ios/`
per CLAUDE.md). Doing that in this iteration would leave the tree in
a half-finished state if the wall clock trips.

Recommendation: deploy TestFlight from the primary
`/Users/pokayoke/Workspace/yaver.io/` checkout (which already has Pods)
after a `git pull` on `autorun/connectivity-mesh-harden/claude` — or merge
this branch to `main` first and deploy from a warm primary tree.
